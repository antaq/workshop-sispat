Integração - Detalhes Práticos

Requisitos técnicos e boas práticas para desenvolvedores

SISPAT

Autenticação

O acesso aos endpoints é protegido e requer o envio da API Key no header da requisição. Utilize o padrão Bearer Token para autorização segura.

Exemplo: Authorization: Bearer {API_KEY}

Content-Type

Todas as requisições de envio de dados (POST/PUT) devem especificar o tipo de conteúdo como JSON. Isso garante que o servidor interprete corretamente o payload enviado.

Header: Content-Type: application/json

Respostas HTTP

O sistema utiliza códigos de status padrão para indicar o resultado das operações:
201 Created: Recurso criado com sucesso.
400 Bad Request: Erro na estrutura ou validação.
401 Unauthorized: Falha na autenticação (chave inválida).

Boas Práticas

• Validação: Verifique o schema JSON antes do envio para evitar erros 400.
• Paginação: Ao enviar grandes volumes, divida em lotes menores para evitar timeouts.
• Idempotência: Trate reenvios em caso de falha de rede sem duplicar registros.
